## What Section 4.1 Says

### Search phase (three explicit steps)

1. **Step 1** — Minimize \(t_w = \sum_{i=0}^{38} H(\Delta W_i)\) subject to nonzero differences only in the 9 message words above. Concrete bit patterns are **not** fixed; only \(t_w\) is found.

2. **Step 2** — Under fixed \(t_w\) and shape constraints:
   - \(\forall i \in [19, 38]: \delta A_i = 0\)
   - \(\forall i \in [23, 38]: \delta E_i = 0\)
   - \(\delta W_i = 0\) outside the 9 words

   Minimize \(t_A = \sum_{i=0}^{38} H(\Delta A_i)\). Do **not** fix the full \((\Delta W, \Delta A, \Delta E)\) trail yet.

3. **Step 3** — Add \(\sum H(\Delta A_i) = t_A\), then minimize \(\sum_{i=0}^{38} H(\Delta E_i)\) and output the full characteristic (Table 4).

### Value transitions

Footnote in §4.2 (referencing 39-step):

> When searching for the differential characteristic for 39-step SHA-256, this strategy [value transitions at certain steps] was **not applied** because we found that the obtained differential characteristic was **valid**.

Message modification uses a **separate** value-transition SAT model to find conforming message pairs (Table 5). That step is outside `find_dc_model_39.py`.

## `find_dc_model_39.py` — What the Code Actually Does

### Closest to the paper among all attack scripts

Unlike the SHA-512 scripts, this file has **no hardcoded `ss` / `ee` / `aa` strings**. The characteristic is found by SAT.

All three paper steps are encoded in **one combined SAT run** via fixed equality constraints plus a descending HW loop on \(\Delta E\):

| Paper step | Code mechanism |
|------------|----------------|
| Step 1 (\(t_w\)) | Hard constraint: \(\sum_{i=8}^{38} H(\Delta W_i) = 25\) on `wd_` variables |
| Step 2 (\(t_A\)) | Hard constraint: \(\sum_{i=8}^{26} H(\Delta A_i) = 10\) on `xd_` variables |
| Step 3 (minimize \(H(\Delta E)\)) | `obj_value()` iteratively minimizes \(\sum_{i=8}^{26} H(\Delta E_i)\) via `yd_` |

```python
# In main() — fixed tw and tA
temp += "0bin000000000@%s) = 0bin%s;\n" % ("wd_" + str(i) + "_" + str(j), bin(25)[2:].zfill(10))
temp += "0bin000000000@%s) = 0bin%s;\n" % ("xd_" + str(i) + "_" + str(j), bin(10)[2:].zfill(10))

# In obj_value() — Step 3
temp += "0bin000000000@%s) = 0bin%s;\n" % ("yd_" + str(i) + "_" + str(j), bin(obj)[2:].zfill(10))
```

### Remaining gaps vs paper

| Paper | Code | Issue |
|-------|------|-------|
| Step 1: **find minimal** \(t_w\) | Fixes \(t_w = 25\) | Values discovered offline, not minimized in-script |
| Step 2: **find minimal** \(t_A\) | Fixes \(t_A = 10\) | Same — not a separate minimization pass |
| Three **sequential** runs with output chaining | Single SAT file with all constraints | Workflow compressed, but semantics preserved |
| Step 3 sums \(i = 0..38\) | Active SAT window: steps **8–26** only | Dense region only; silent tail via `op2`/`op5` |
| Separate value-transition correction | No `correct_dc` script | **Consistent with paper** — DC valid without it |

### Other code details

- `start_step = 8`, `end_step = 27` — differential constraints on rounds 8–26.
- `op6` all **1** (Method 1 / expansion mode for message words \(i > 15\)).
- `op7` all **0** — no value-transition model inside `find_dc` (matches paper footnote).
- `init_HW = 90` — starting bound for descending search on \(H(\Delta E)\).

## No `correct_dc` Script — Expected

The paper states the 39-step SHA-256 differential characteristic was **valid without** adding value-transition constraints during the DC search. The repo reflects this: only `find_dc_model_39.py` exists.

Message-pair verification (Table 5) uses value transitions in a **different** phase (described in §4.1 “Message modification”), not as a `correct_dc_*` script.

## Summary Verdict

| Aspect | Match? |
|--------|--------|
| Message word set | **Yes** |
| Cancellation boundaries (`op2`/`op5`) | **Yes** |
| Three-step **logic** (fix \(t_w\), fix \(t_A\), minimize \(H(\Delta E)\)) | **Mostly yes** — best fidelity in the whole codebase |
| Three-step **workflow** (separate runs, iterative minimization) | **No** — single run with precomputed \(t_w=25\), \(t_A=10\) |
| Hardcoded Table 4 | **No** — trail is searched |
| Value-transition correction script | **N/A** — correctly absent |
| Automated pipeline to collision verifier | **No** — manual / separate tooling |

**Bottom line:** The 39-step SHA-256 script is the **most faithful** implementation of the paper's search procedure. The main simplification is collapsing three sequential minimization passes into one SAT instance with pre-determined \(t_w\) and \(t_A\), rather than discovering those minima inside the script.
